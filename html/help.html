Introduction

The Northeast Aquatic Connectivity results and custom analysis tool are designed to allow users to assess the relative benefits of removing barriers to fish and other aquatic organisms. The tool outputs a list of barriers prioritized based on the user's inputs. The prioritized results are not a prescription for action and, importantly, do not incorporate any information on social factors, economics, feasibility or opportunity. Further, they are agnostic of any particular action and could include dam removals, culvert upgrades, or other passage improvments.

Exploring the Consensus Results

You can use the controls on the "Explore Consensus Results" tab to filter the prioritized results and download the consensus scearnio results as an ArcGIS geodatabase.

Sliders on this dialog allow the consensus resutls to be filtered based on their prioritized result tier, degree of passability (e.g. severe barrier, moderate barrier, etc) or both. Alternately, a customized filter can be applied to limit the results to a state, watershed, or other attribute.

The "Download Consensus" button will start a download of a zipped ArcGIS geodatabase with the consensus anadromous results and a .lyr file for symbology.

Optional: Filters

The universe of barriers that are assessed in the tool can be restricted by applying an optional filter to the analysis. For example, the analysis can be limited to one or more subwatersheds, towns, or counties. To apply a filter, check the "Filter Barriers?" option, select an attribute to filter by, select an operator (e.g. "=") and finally select a value. If multiple are be selected be sure to use the "IN" operator. More information on uilding ArcGIS-compliant queries can be found here

Applying relative weights

Relative weights are applied to metric (metric weights must total 100) to indicate how important that metric is for the objectives of the prioritization. For example, if opening the most upstream habitat is the sole objective of a prioritization, a weight of 100 can be applied to the "Upstream functional network length" metric which measures the connected river habitat upstream of each barrier. In this scenario, the barrier with the most connected habitat upstream would be the top priority. The more weight that is applied to a metric, the more bearing it will have on the results. A metric with zero weight has no bearing on the results. Each metric name in the tool is a link whcih leads to a description of the metric and the data that was used to generate it.

Metrics are grouped in tabs by theme. These groupings are provided to organize the metrics in a logical framework; the groupings themselves do not impact the prioritization.

Consensus weights have been developed by the NAACC Northeast Aquatic Connectivity subcommittee for both a anadromous fish prioritization scenario. Clicking the "Anadromous" button will populate the tool with the appropriate weights.

Optional: barriers to remove

A set of up to ten barriers can be selected and the analysis run as if these barriers did not exist on the landscape. This can be done to assess the impacts of a proposed removal on the surrounding priorities and to inform a better understanding of the potential gains of mitigating multiple barriers in series.

To select barriers for removal, check the "Model Barrier Removal?" box. This will expand a text box and load a temporary map layer with all barriers symbolized as black points. Clicking on one of these barriers will select it and populate the text box with its ID properly formatted with single quotes and comma separation. Up to 10 barriers may be selected for removal. Alternately, a list of barrier SiteIDs formatted with single quotes and comma separators, can be typed into the text box directly.

Optional: summary statistics

Optionally, summary statistics can be run on the prioritization results. Using this functionality it is possible to ascertain, for example, the average barrier priority within a subwatershed. To use this functionality, check the "Calculate Summary Statistics?" box. This will reveal drop down menus to select a unit to summarize within (e.g. Town, Watershed) and a value to summarize (e.g. result tier).

Using the Results

When an analysis is complete, the results will be added to the map colored by their priority based on the metrics and weights selected by the user. Warm colors are the higher Tiers while cool colors are lower Tiers. A Tier 1 priority barrier (red) is a top priority for improving aquatic organism passage while a Tier 20 barrier (blue) is a low priority.

In addition to the result map layer, when an analysis is complete, the tool interface switches to the "Results" screen populated with sliders that allow users to filter the results based on result Tier or barrier severity. Clicking on a result barrier raises a popup window with the attributes and results for that barrier. If barriers were selected for removal, the values in the popup will reflect the removal of the selected barriers. For example, the "Count of Downstream Barriers" will be one less for all barriers upstream of a removed barrier.

The consensus anadromous & custom results can be toggled on and off using the "Display Consensus Anadromous Scenario?" and "Display Results?" checkboxes at the top of the tool. At the top of the tool, are radio buttons to switch between the inputs screen, the results screen, and the summary statistics results, if summary statistics are run.

The bottom of the results screen includes buttons to downlaod the analysis inputs, the custsom analysis results (as an ArcGIS geodatabase) and to access this help.
